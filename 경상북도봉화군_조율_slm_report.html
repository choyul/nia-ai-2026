01모델 선택 이유

경상북도 봉화군 도시계획과 도시재생팀 업무 환경에서 활용 가능한 소형 언어 모델(SLM) 후보로 본 모델을 선택함.

선정 사유는 다음 4가지임.

○ 공공기관 활용 시 라이선스 제약이 가장 적음 : Apache 2.0 라이선스로 상업적·비상업적 활용 모두 자유로움

○ 소규모 인프라에서 구동 검증 가능 : 1.5B 파라미터 규모로 GPU가 없는 일반 사무용 PC 환경에서도 양자화 모델 형태로 실행 가능

○ 한국어를 포함한 29개 언어 공식 지원 : 행정 문서 처리·민원 응대 등 한국어 기반 업무 검증에 적합

○ 장문 컨텍스트 처리 능력 : 최대 32,768 토큰 입력 지원으로 긴 보고서·법령·매뉴얼 처리 실험에 부합

본 보고서는 모델 자체의 기술 사양을 객관적으로 분석하는 것이 목적이며, 향후 실제 행정 도입 여부는 추가 검증을 거쳐 별도 판단할 사안임.

03학습 데이터와 튜닝 방식

사전학습은 알리바바가 자체 구축한 대규모 코퍼스 약 18조(18 trillion) 토큰을 활용하였음. 다만 데이터셋의 세부 출처·구성 비율은 모델 카드와 공식 블로그에서 상세히 공개되지 않음.

튜닝 방식은 다음 두 단계로 명시되어 있음.

○ 사전학습(Pre-training) : 코딩·수학 특화 expert 모델을 활용한 대규모 사전학습으로 지식 밀도 향상

○ 사후학습(Post-training) : 지도학습 기반 추가 훈련(SFT·Supervised Fine-Tuning)에 더해 사람 선호도 기반 정렬 기법(DPO·Direct Preference Optimization 등)이 적용된 것으로 명시됨. 명령 수행, 장문 생성(8천 토큰 이상), 표·JSON 등 구조화 출력 능력이 강화됨

한국어 데이터 비중 및 한국 행정 도메인 특화 데이터 포함 여부는 공개 범위 내에서는 확인이 어려움. 한국어가 공식 지원 29개 언어 중 하나로 명시되어 있으나, 학습 코퍼스 내 한국어 비율은 비공개 사항임.

메모리 사용량을 줄인 경량화 모델(양자화 모델)도 여러 포맷(GPTQ·AWQ·GGUF 등)으로 동일 조직(Qwen) 산하에서 별도 제공되고 있어, GPU 메모리 제약이 큰 환경에서는 경량화 모델 활용을 검토할 수 있음.

06활용 가능 업무

일반 행정 업무 환경에서 다음 5개 영역의 활용 가능성이 검토되며, 각 영역별 실행 절차는 다음과 같음.

1. 장문 행정문서 요약

○ 적용 대상 : 사업계획서·실시설계서·연구용역 보고서 등 30~50쪽 분량 문서

○ 실행 절차

- 입력 전 PDF·HWP·Word를 텍스트로 추출

- 장문은 적정 단위로 분할(chunking)하여 관련 부분만 모델에 입력 (32K 컨텍스트 지원이지만 1.5B 모델은 후반부 정보 누락 가능성)

- JSON·표 형태 구조화 출력으로 받아 결재문서에 그대로 활용

2. 민원 응대 1차 초안 작성

○ 실행 절차

- 민원 본문 입력 전 개인정보(성명·주민번호·연락처·주소)를 마스킹 처리

- 모델은 응답 초안만 생성하고, 대민 발송은 자동화하지 않음 — 담당자 검토·승인 필수

- 빈번 민원 유형은 사전 분류 → 유형별 응답 템플릿과 결합

3. 회의록·간담회 자료 정리

○ 32K 토큰 입력 활용으로 장시간 녹취 텍스트를 단일 처리 단위로 요약

○ 안건별 발언·결정사항·후속조치 항목을 표 형태로 추출

4. 내부 지식 검색 (RAG 결합)

○ 실행 절차

- 과거 사업 보고서·내부 규정·결재문서를 의미 검색이 가능한 형태로 사전 처리하여 검색용 저장소(벡터 데이터베이스 등)에 등재

- 사용자 질의에 대해 관련 문서 단락을 검색 → 검색된 근거 단락만 모델에 전달

- 출처 문서명·페이지·근거 문장을 응답과 함께 표시하여 환각 위험 통제

5. 번역 및 다국어 응대 지원

○ 영문 사례집·해외 정책자료 1차 번역 및 외국인 주민 대상 행정 안내 초안

○ 최종 발송 전 행정 용어 정확도는 담당자 재검토 필수

위 5개 영역 모두 단독 모델만으로는 한계가 있으며, RAG 결합·입력 정제·담당자 검수의 3중 절차가 전제됨.

07한계와 주의사항

행정 적용 관점에서 다음 7개 한계가 식별됨.

1. 한국어 성능 한계

○ 모델 카드에 한국어가 지원 언어로 명시되어 있으나, 학습 데이터의 한국어 비중과 한국 행정 도메인 데이터 포함 여부는 공개되지 않음

○ 일부 사용자 보고에 따르면 시스템 프롬프트 미설정 시 한국어 응답에 중국어 토큰이 혼입되는 사례 있음

2. 환각(Hallucination) 위험

○ 1.5B 소형 모델 특성상 사실 관계 추론 능력이 제한적이며, 모르는 정보를 그럴듯하게 만들어내는 경향이 더 큼

○ 행정 적용 시 RAG 결합으로 출처 문서를 함께 제공하여 사실 근거를 강제할 필요 있음

3. 장문 컨텍스트 활용 제약

○ 32K 토큰 입력은 가능하나, 1.5B 모델 특성상 장문 입력 시 중간 부분 정보 누락 현상(이른바 lost-in-the-middle 문제) 가능성

4. 추론 능력 한계

○ 복잡한 다단계 추론·법령 해석·정책 의사결정 등은 7B 이상 상위 체급 모델이 권장됨

○ 1.5B는 단순 질의응답·요약·분류·초안 생성 보조 도구로 한정 적용

5. 프롬프트 인젝션(Prompt Injection) 위험

○ chat template의 특수 토큰(<|im_start|>, <|im_end|> 등) 문자열이 사용자 입력이나 첨부 문서 본문에 포함될 경우 프롬프트 구조를 교란하여 의도하지 않은 출력 발생 가능

○ 입력 정제(input sanitization) 및 특수 문자 처리(escaping) 정책의 별도 설계가 필요함

6. 응답 일관성 통제 필요

○ generation_config의 기본값이 do_sample=true로 설정되어 있어 동일 질의에도 응답이 매번 달라질 수 있음

○ 행정·민원 답변의 일관성이 요구되는 업무에서는 응답 다양성을 낮추는 방향(temperature 하향, do_sample 비활성화 등)과 출처 강제 프롬프트 적용 등 통제 방안의 검토가 필요함

○ Qwen 공식 권장 추론 파라미터: temperature=0.7, top_p=0.8, top_k=20 (창작·대화용 기준)

7. 라이선스 외 운영 리스크

○ Apache 2.0 자체는 자유롭지만, 모델 출력물의 정확성에 대한 책임은 운영기관에 귀속됨

○ 행정 의사결정에 직접 반영 시 면책·검증 절차 별도 설계 필요

08종합 의견

Qwen2.5-1.5B-Instruct는 일반 행정 업무 환경에서 소형 모델 시범 도입 후보로서 다음 3가지 강점을 가짐.

○ Apache 2.0 라이선스에 따른 도입 절차 단순성

○ 1.5B 규모로 인한 인프라 진입 장벽 최소화

○ 한국어 지원 및 장문 컨텍스트 처리 능력

다만 본격 도입을 검토하는 경우 다음 3단계 추가 검증이 필요함.

1. 1단계 : 한국어 응답 품질 베이스라인 측정

○ 행정 분야 표준 질의 30~50문항으로 응답 정확도·자연스러움·환각률 측정

2. 2단계 : 활용 시나리오별 PoC

○ 요약·분류·질의응답 등 업무 유형별 적용 시 성능 비교

3. 3단계 : 비교군 검증

○ 동일 시나리오에서 한국어 특화 모델(EXAONE 3.5-2.4B 등) 및 상위 체급 모델(Qwen2.5-7B 등)과 비교 후 적정 모델 선정

1.5B 규모는 결과물 품질 자체보다는 소형 모델의 행정 적용 가능성을 가늠하는 학습용·실험용 모델로서 가치가 있다고 판단함.

한편 본 모델은 행정 RAG 시스템에서 '응답 생성' 자리에 해당하는 카테고리임을 명시할 필요가 있음. 일반적으로 행정 환경의 RAG 기반 AI 시스템은 다음 3개 자리로 분리되어 각각 별도 모델군의 검토가 필요함.

자리역할검토 대상 모델군
① 응답 생성
(Generation)		사용자 질의에 자연어 답변 생성생성형 LLM (본 보고서 분석 범위) 또는 클라우드 LLM API
② 의미 검색
(Embedding)		문서를 검색 가능한 벡터 형태로 사전 처리한국어 지원 임베딩 모델 (BGE 계열·multilingual-e5 계열 등 별도 검토 필요)
③ 입력 정제
(Preprocessing)		개인정보 탐지·마스킹 등 전처리분류형 SLM 또는 정규식 기반 처리

따라서 본 보고서의 분석 범위는 ① 응답 생성 자리에 한정되며, 본격 도입 시점에는 ② 의미 검색용 임베딩 모델과 ③ 입력 정제 처리 방식에 대한 별도 검토와 보고가 필요함. 1회차 본 보고서는 RAG 시스템 모델 선택 관점에서 응답 생성 카테고리의 학습 단계에 해당함.
